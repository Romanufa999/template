# UI Psychology — Психология интерфейсов и микровзаимодействий

Гайдлайн по применению психологии геймификации, микровзаимодействий и нейронауки вовлечения в лэндингах. Цель — удерживать внимание от секции к секции и мотивировать мозг взаимодействовать с сайтом и формами.

**ВАЖНО:** Этот навык применяется АВТОМАТИЧЕСКИ при создании/редактировании любого лэндинга. Не ждать запроса от пользователя — встраивать эти принципы в каждую секцию, форму и переход.

---

## 1. Модель Hook (Нир Эяль) — петля вовлечения

Каждая секция лэндинга — это мини-цикл «Триггер → Действие → Награда → Инвестиция»:

| Этап | Как применять на лэндинге |
|------|--------------------------|
| **Триггер** | Scroll-triggered анимация, пульсирующий бейдж, мигающий индикатор — внимание захвачено |
| **Действие** | Клик по табу, выбор типа недвижимости, скролл таблицы, перемещение слайдера — минимальное усилие |
| **Переменная награда** | Контент раскрывается нелинейно: карточки появляются с разной задержкой, числа анимированно считают, SVG «оживают». Ключ — **непредсказуемость** момента награды |
| **Инвестиция** | Пользователь уже выбрал тип, двигал слайдер, скроллил таблицу — теперь форма кажется логичным шагом, а не навязыванием |

### Обязательные реализации:
- Каждая секция должна содержать хотя бы одно **интерактивное действие** до формы (таб, слайдер, карусель, сортировка)
- После действия — **визуальная награда** (анимация, смена контента, появление нового блока)
- Форма всегда идёт **после** интерактива, а не вместо него

## 2. Микровзаимодействия — «сочность» (Juicy Design)

Принцип из геймдизайна: каждое действие пользователя должно давать **преувеличенный, удовлетворяющий отклик**. Мозг воспринимает это как «среда откликается на меня» → дофаминовое подкрепление → желание продолжать.

### Обязательный набор для каждого проекта:

#### a) Кнопки
- **Hover**: lift + scale(1.02) + shadow expansion, `cubic-bezier(0.34, 1.56, 0.64, 1)` (spring)
- **Press**: scale(0.97) + shadow compress + мгновенный transition (0.08s)
- **Submit success**: scale → bounce + цвет меняется + появляется галочка с анимацией
- **Disabled**: `cursor-not-allowed` + приглушённый цвет + без hover-эффектов

#### b) Инпуты
- **Focus**: scale(1.01) + glowing ring расширяется пружинно + placeholder сдвигается
- **Typing**: каждый символ — subtle border-color pulse (опционально)
- **Validation success**: зелёная тень + появление иконки check

#### c) Карточки
- **Hover**: translateY(-4px) + scale(1.005) + shadow intensify + border подсветка
- **Entrance**: staggered fadeInUp с задержкой 60–120ms между карточками
- **На мобиле**: swipe-to-reveal вместо hover

#### d) Табы / фильтры
- **Active**: spring-transition на фон + тень
- **Press**: scale(0.95) мгновенно → spring назад
- **Switch**: content fade-out/fade-in с translate для ощущения перемещения

#### e) Слайдеры
- **Drag thumb**: scale(1.2) + glow ring
- **Value change**: label обновляется с number counter анимацией

#### f) SVG иконки внутри кнопок
- **Hover**: scale(1.08) с spring easing
- **Active**: scale(0.92)
- **Arrow icons**: translateX(3px) on hover

### CSS-основа (добавлять в globals.css каждого проекта):
```css
/* Spring easing for micro-interactions */
:root {
  --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);
  --ease-ios: cubic-bezier(0.32, 0.72, 0, 1);
}

/* Buttons: hover lift + press depth */
button[type="submit"] {
  transition: transform 0.2s var(--ease-spring),
              box-shadow 0.3s ease,
              background-color 0.2s ease;
}
button[type="submit"]:hover {
  transform: translateY(-1px) scale(1.02);
}
button[type="submit"]:active {
  transform: translateY(1px) scale(0.97);
  transition-duration: 0.08s;
}

/* Input focus glow */
input:focus {
  transform: scale(1.01);
  box-shadow: 0 0 0 4px rgba(var(--accent-rgb), 0.12);
}

/* Placeholder shift on focus */
input:focus::placeholder {
  opacity: 0.4;
  transform: translateX(4px);
}
```

## 3. Удержание внимания от секции к секции

### a) Эффект Зейгарник — незавершённые задачи
Мозг запоминает незавершённые действия лучше, чем завершённые. Каждая секция должна **намекать на следующую**:
- Визуальный «хвост» секции уходит за видимую область → пользователь скроллит
- Текст-тизер внизу секции: «А ещё есть скрытые скидки...» → следующая секция раскрывает
- Пронумерованные шаги (01, 02, 03) создают ощущение незаконченной последовательности

### b) Прогрессивное раскрытие
- Контент появляется **только при скролле** (IntersectionObserver с threshold 0.1–0.3)
- Staggered entrance: элементы появляются каскадом, не все сразу
- Числа анимируются от 0 до значения (counter animation) — мозг следит за процессом
- Прогресс-бар или dots indicator — показывает сколько секций осталось (опционально)

### c) Pattern interruption — перезапуск внимания
Внимание падает каждые 8–10 секунд. Между секциями нужны «перебивки»:
- Смена цветовой схемы (светлая → тёмная → светлая)
- Смена layout-а (таблица → карточки → форма → SVG-иллюстрация)
- Неожиданный интерактивный элемент (3D-тилт, карусель, видео-превью)
- Микро-анимации «жизни»: пульсирующие точки, бегущие волны, мерцание

### d) Визуальная связность между секциями
- SVG-волны или gradient fade на стыке секций — плавный переход
- «Энергетические линии» (animated path с traveling dots) между шагами процесса
- Единая цветовая нить: accent color появляется в каждой секции, но в разной дозировке

### Обязательные реализации:
```
Секция 1: Hero — эмоция + CTA-тизер
  ↓ (gradient fade / wave divider)
Секция 2: Социальное доказательство — числа, логотипы, кейсы
  ↓ (смена цвета фона)
Секция 3: Интерактив — табы/фильтры/слайдер (пользователь ДЕЙСТВУЕТ)
  ↓ (pattern interruption — SVG-иллюстрация)
Секция 4: Углубление — таблица/детали (скролл внутри секции)
  ↓ (тизер-текст → «получите персональное предложение»)
Секция 5: Форма — после инвестиции усилий конверсия выше
```

## 4. Психология форм — конверсионные микровзаимодействия

### a) Foot-in-the-door
Пользователь уже сделал 3–5 микро-действий на странице (кликал табы, двигал слайдер, скроллил таблицу). Психологически он **инвестировал** время → не хочет терять «прогресс» → форма — логичный следующий шаг.

**Правило**: Форма НИКОГДА не должна быть первым взаимодействием на странице. Перед формой — минимум 2 интерактивных элемента.

### b) Визуальная награда за заполнение
- **Фокус на инпут**: поле «расцветает» — glow ring пружинно расширяется
- **Ввод номера**: при наборе 11 цифр кнопка «оживает» — меняет цвет, появляется тень, subtle pulse
- **Submit**: Кнопка → bounce → цвет меняется → текст «Заявка отправлена» с иконкой ✓
- **Форма после submit**: зелёная рамка / glow — визуальная «награда»

### c) Обрамление формы
- Форма визуально выделена: рамка с анимированными dash-линиями, gradient glow при hover
- Proximity principle: рядом с формой — trust-сигналы (иконка щита + «Не спам», «Только личные клиенты»)
- Urgency: если уместно — «Осталось X дней», «3 человека смотрят сейчас»

### d) Анимация формы при входе в viewport
- Форма «пульсирует» при первом появлении на экране (привлечение внимания)
- Задержка 0.3–0.5s перед анимацией — мозг успевает заметить элемент до его «оживления»

### Обязательная CSS-анимация для формы:
```css
@keyframes formPulseEntry {
  0%   { box-shadow: 0 0 0 0 rgba(var(--accent-rgb), 0); }
  20%  { box-shadow: 0 0 25px 6px rgba(var(--accent-rgb), 0.3); }
  40%  { box-shadow: 0 0 0 0 rgba(var(--accent-rgb), 0); }
  60%  { box-shadow: 0 0 20px 4px rgba(var(--accent-rgb), 0.2); }
  100% { box-shadow: 0 0 0 0 rgba(var(--accent-rgb), 0); }
}
```

## 5. Геймификация — игровые механики в неигровом контексте

### a) Коллекционирование (Collection)
Выбор типов недвижимости, фильтров, городов — ощущается как «собирание предметов»:
- Прогресс-бар показывает «Выбрано 3/5»
- Каждый выбор — визуальная награда: particle burst, badge pop, glow
- Selected items визуально отличаются: белая иконка, галочка-бейдж, подсветка

### b) Прогресс и разблокировка (Unlock)
- Секции «разблокируются» при скролле — opacity 0 → staggered appear с эффектом unlock
- Шаги процесса (01 → 02 → 03) с energy beam между ними, заполняющимся при скролле
- Progress ring вокруг иконки шага — рисуется при появлении в viewport

### c) Streak / Combo
- При быстром скролле через несколько секций — escalating visual reward
- Путешествующие точки по SVG-линии между секциями — скорость зависит от scroll momentum
- «Живые» элементы на фоне (волны, пульсы, мерцание) усиливаются при активном взаимодействии

### d) Particle effects
Для ключевых действий (выбор, submit, toggle):
```css
@keyframes particleBurst {
  0%   { transform: translate(-50%, -50%) scale(0); opacity: 1; }
  100% { transform: translate(
    calc(-50% + cos(var(--angle)) * 20px),
    calc(-50% + sin(var(--angle)) * 20px)
  ) scale(0); opacity: 0; }
}
```

### e) Ripple / Wave on click
Каждый клик по интерактивному элементу — волна расходится от точки нажатия.

## 6. Нейронаука вовлечения

### a) Дофаминовая антиципация
Награда — не в самом событии, а в **ожидании**. Применение:
- Skeleton-loading перед появлением контента (0.3–0.5s) → контент «разблокируется»
- Числа считают от 0 до значения (не появляются сразу)
- Hover-состояние показывает превью действия до клика

### b) Эффект изоляции (Von Restorff)
Элемент, отличающийся от окружения, запоминается лучше. CTA-кнопка и форма должны:
- Контрастировать по цвету с окружением
- Иметь уникальную форму/размер среди соседних элементов
- Быть единственным «ярким» пятном в зоне видимости

### c) Цветовая психология
- **Зелёный (Sber green)**: доверие, безопасность, рост → для CTA и success-состояний
- **Тёмный фон**: фокус, премиальность, контраст для светлых элементов
- **Тёплые тона**: уют, эмоциональная близость → для форм и testimonials
- **Красный/оранжевый (дозировано)**: urgency, дедлайны, обратный отсчёт

### d) Когнитивная лёгкость (Cognitive Fluency)
Чем легче воспринять — тем больше доверия:
- Плавные анимации (ease-out, spring) вместо резких (linear)
- Закруглённые углы (>12px) воспринимаются безопаснее, чем острые
- Единообразие: одинаковые spring-easing по всему сайту → подсознательная предсказуемость

## 7. Технический чеклист — обязательно для каждого проекта

### В globals.css:
- [ ] CSS-переменные `--ease-spring` и `--ease-ios`
- [ ] Глобальные micro-interactions: кнопки (hover/active), инпуты (focus/placeholder), карточки (hover/entrance)
- [ ] Scroll reveal: `.reveal-on-scroll` + `.reveal-stagger` с IntersectionObserver
- [ ] Form pulse entry animation
- [ ] SVG contour animations (draw, pulse, wave, sway)
- [ ] Scrollbar hide для кастомных скроллов

### В каждой секции:
- [ ] IntersectionObserver для scroll-triggered появления (threshold 0.1–0.3)
- [ ] Staggered entrance элементов (delay 60–120ms между ними)
- [ ] Минимум 1 интерактивный элемент (таб, слайдер, карусель, сортировка)
- [ ] Визуальная связь с предыдущей/следующей секцией (gradient, wave, энергетическая линия)
- [ ] Смена layout-паттерна относительно соседних секций

### В формах:
- [ ] Form pulse при входе в viewport
- [ ] Input glow на focus с spring easing
- [ ] Кнопка disabled при невалидном вводе → активируется с визуальным «оживанием»
- [ ] Success state с bounce + текст + иконка
- [ ] Trust-сигнал рядом с формой

### Между секциями:
- [ ] Чередование светлого/тёмного фона или gradient transitions
- [ ] Pattern interruption каждые 2–3 секции (смена типа контента)
- [ ] Тизер следующей секции виден при скролле

## 8. Чего НЕЛЬЗЯ делать

- **Не перегружать**: максимум 2–3 типа анимации на экран одновременно
- **Не блокировать скролл**: никаких fullpage.js, scroll hijacking
- **Не замедлять**: анимации > 700ms ощущаются тормозом. Entrance: 300–600ms. Hover: 150–300ms. Active: 60–100ms
- **Не анимировать layout properties**: только transform и opacity (GPU-ускоренные)
- **Не повторять одну анимацию**: если все секции появляются одинаково — эффект теряется
- **prefers-reduced-motion**: уважать системные настройки пользователя
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```
