# Техзадание на сайт — конвейер

Пайплайн из 5 шагов. Результат — ТЗ = машина убеждения, НЕ информационная брошюра.

## Философия

**Три столпа ТЗ:**
1. **Убеждение** — каждая секция работает на конкретный психологический паттерн (social proof, authority, reciprocity, loss aversion, anchoring). Паттерн «Почему мы» НЕ отдельная секция — trust_triggers рассыпаны по ВСЕМ секциям.
2. **Удержание** — каждая секция визуально отличается от предыдущей (другой layout, фон, ритм). Минимум 2 микровзаимодействия на секцию. Разнообразие = интерес = дочитывание до конца.
3. **Захват** — минимум 4 точки захвата на странице с разными CTA-текстами. Калькулятор/квиз = форма. Лид-магнит для холодных. Мессенджеры как альтернатива.

**Конкуренты НЕ эталон.** Даже если все слабые, ТЗ на уровне лучших сайтов в digital. Анализ конкурентов — входные данные, не шаблон.

## Подскиллы

При активации этого скилла автоматически загружай следующие подскиллы:

| Подскилл | Описание | Когда применять |
|----------|----------|-----------------|
| `Анализ конкурентов и ниши` | Шаг 0. Глубокий анализ конкурентов, ниши и аудитории по 10 методикам | Всегда — первый шаг конвейера |
| `Определение структуры секций` | Шаг 1. Первичная структура секций на основе анализа | Всегда — после шага 0 |
| `Анализ тенденций маркетинга` | Шаг 2. Тренды маркетинга и веб-дизайна в нише | Всегда — после шага 1 |
| `Корректировка структуры` | Шаг 3. Доработка секций на основе трендов | Всегда — после шага 2 |
| `Заливка структуры в базу` | Шаг 4. Запись финальных секций в site_sections | Всегда — финальный шаг |

Подскиллы загружаются из таблицы `sub_skills` по `skill_id` этого скилла.
Если подскилл имеет `status = 'active'` — он обязателен.

## Таблицы
- **projects** — проект, current_step, status
- **project_data** — результат каждого шага (JSONB)
- **site_sections** — финальное ТЗ (12 полей + promt автосборка)

## Сборка promt
Из ЗАПОЛНЕННЫХ колонок: `{N}. {колонка}: {значение}.` Пустые пропускаются.
